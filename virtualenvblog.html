Setting up Virtual Environment for a Django Project

In this blog, we are going to see what is a virtual environment and it's advantages, and how to set it up in Ubuntu for a Django project.

Virtual Environment:

A virtual environment can have different versions of libraries, packages, and interpreters which are isolated from other virtual environments or global environment.A virtual environment is a directory tree which contains Python executable files and other files which indicate that it is a virtual environment.

Why Virtual Environment?

By default, every project on our system will use these same directories to store and retrieve packages (third party libraries). Imagine a scenario where we have to develop two applications and we have Django 2.x and Django3.x installed globally. In such a case, it would be difficult for python to differentiate the packages as they are residing under the same directory. So we create a different environment for each project so that it won't consider looking for any globally installed packages as it checks only the packages that are installed in the virtual environment. In this way, the dependencies will be isolated from the global (system) environment.

Steps for setting up Virtual Environment

  1. Installing virtualenv module

    We use a module called virtualenv. We can install it by the following command

    pip install virtualenv

  2. Creating Virtual Environment

    When we create virtual environment it copies all python binary files from actual system directory to current directory

    virtualenv env_name

    * Here env_name is the environment name, you can choose any name

  3. Choosing python version to be used in virtual environment

    When we have more than one version of python installed on our machine then we can choose which version to be used in virtual environment by the following command

    virtualenv -p /usr/bin/python3.8 env_name

    In this case we have choosed python3.8

  4. Activating Virtual Environment

    To use the virtual environment that we just created we need to activate it by the following command

    source env_name/bin/activate

    Now this command will take us to our virtual environment

  5. Installing Required packages

    We use pip to install the required packages

    pip install django

  6. Checking installed packages

    In order to check the packages installed we can use the following command.

    pip freeze

    This command will list all the packages that were installed.

  7. Exiting from virtual environment

    In order to exit from the virtual enviroment we can use the following command.

    deactivate

I covered the basics as well as the commands required for creating virtual environment for a python project.
